## Design Context

### Users
- Target audience is still being defined; the current "EcoMap" branding and site-selector concept are placeholders for rapid prototyping.
- The interface should feel accessible to a general audience, not gated behind technical jargon or enterprise conventions.
- Users are likely exploring or selecting from a set of locations/items — the job-to-be-done is discovery + decision, not deep data analysis.

### Brand Personality
- **Playful, approachable, confident**
- Voice is encouraging and human — think "friendly guide" rather than "system administrator"
- Emotional goal: make the user feel capable and delighted, not instructed or managed
- The Duolingo-inspired system captures this well: rounded geometry, generous whitespace, bold color blocking, and satisfying micro-interactions

### Aesthetic Direction
- **Visual tone:** Vibrant, clean, and energetic. White/off-white surfaces with bold green (`#58CC02`) and cyan (`#1CB0F6`) accents.
- **References:** Duolingo's UI — chunky buttons with 3D shadows, rounded cards, friendly typography (Plus Jakarta Sans as fallback for Din Round), dot-grid backgrounds that feel playful without being childish.
- **Anti-references:** Corporate enterprise dashboards, dense data tables, gray-on-gray palettes, sterile layouts, generic Bootstrap/Tailwind default aesthetics.
- **Theme:** Light mode first. Dark mode can be explored later but is not a priority.

### Design Principles
1. **Playful but purposeful** — Every animation, color, and rounded corner should serve the experience, not decorate it. Avoid novelty for novelty's sake.
2. **Generous whitespace** — Follow the 8px spacing scale strictly. Breathing room reduces cognitive load and reinforces the calm, approachable tone.
3. **Bold hierarchy through weight and color** — Use 700/800 weights and brand green for primary actions. Never rely on size alone.
4. **Rounded = friendly** — Default to `12px` border-radius on cards and buttons, `16px` on larger surfaces, `9999px` only for avatars/badges.
5. **Shadows signal life** — Resting surfaces are flat or Level 1 (`0px 1px 3px`). Interactive elements gain elevation on hover. Buttons use the signature "Duo shadow" (`0 4px 0 #46A302`) for tactile feedback.
6. **Accessibility by default** — Minimum 44×44px touch targets, focus rings on all interactives (`0 0 0 3px rgba({color}, 0.15)`), and never use color alone to convey status.
